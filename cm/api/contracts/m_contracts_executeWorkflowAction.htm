POST /api/contracts/{ContractID}/actions

Executes workflow actions associated with a specific change of contract status.

Authorization Roles/Permissions: To complete this operation successfully, a user must be an administrator for the applicable app or API. However, in addition, the specific workflow action to be executed must be valid for the specific contract the action will be performed on. For more information, see Executing Workflow Actions.

Sample Request

The example below shows an API access request being approved.

Request URL

https://<hostname>/api/contracts/contract25519.acmecorp/actions

Sample request headers

POST /api/contracts/contract25519.acmecorp/actions HTTP/1.1
Host: <hostname>
Accept: */*
Content-Type: application/json; charset=UTF-8

Sample request body

{
  "ActionName":"apicontract.action.activate",
  "Comments":"Granting production access (activating)."
}

Request Parameters

Parameter Parm Type Data Type Required Description
ContractID Path string Required The unique ID for the App/API contract.
Action Body Action Required Contains information about an action performed as part of a workflow-related activity. For information on possible values for groups, see All Groups: Valid Workflow Actions. For app/API contracts, see App/API Contracts: Valid Workflow Actions. For tickets, see Tickets: Valid Workflow Actions. ActionName is required.

Response

If successful, this operation returns HTTP status code 200, with the new contract status in plain text.
