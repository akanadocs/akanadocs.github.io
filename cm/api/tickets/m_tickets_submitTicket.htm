POST /api/tickets

Opens a new ticket.

Authorization Roles/Permissions: The user must be logged in, and must be an authorized user for the resource the ticket relates to.

Sample Request

The example below shows a request to add a specific ticket.

Request URL

https://<hostname>/api/tickets

Sample request headers

POST /api/tickets HTTP/1.1
Host: <hostname>
Accept: text/html, */*; q=0.01
Content-Type: application/json; charset=UTF-8

Sample request body

{
  "UserID":"user12262.acmecorp",
  "Description":"When I try to use more than one payment type the transaction hangs up. It's happened to me twice.",
  "Title":"Bug when using multiple payment types",
  "Environment":"Production",
  "APIID":"apiversion20693.acmecorp",
  "AppID":"4iXro2Uxs2h4LviCshR9pK9w.acmecorp",
  "Label":[
    "com.soa.ticket.label.priority.high"
  ],
  "Tag":[
    "bug",
    "payment",
    "multiple"
  ]
}

Request Parameters

Parameter Parm Type Data Type Required Description

Response

If successful, this operation returns HTTP status code 200, with the TicketID for the new ticket.
